How to Use the AFA Tracker

Getting Started: First-Time Setup

Before you can use the calculator, you must load the OHIP Fee Schedule. This file contains all the official billing codes and their values.

  • Click the "Load OHIP Fee Schedule File" button.
  • Select the master file from your computer (usually named `OHIPMASTER.001` or similar).
  • The app will parse the file and save it to your browser's local storage. This is a one-time setup. You won't need to load the file again unless it's updated or you clear your browser's storage.

Managing Shifts

All of your shifts are stored securely in your browser's database. No more managing individual files!

  • Creating a New Shift: Select "-- Create New Shift --" from the "Load Existing Shift" dropdown. The "Shift Date" field will become active. Choose a date, select the correct "Shift Type", and begin adding patients.
  • On Call Shifts: Select "On Call" from the shift type dropdown. Enter the hours worked in the respective fields (Day, Evening, Night, Weekend) to calculate base pay automatically.
  • Loading an Existing Shift: Simply select a date from the "Load Existing Shift" dropdown to view or edit its data.
  • Saving a Shift: After entering or editing data, click the "Save Current Shift" button. This will save the shift to your local database. If you try to save a new shift on a date that already has a saved shift, the app will ask you to confirm before overwriting.
  • Changing a Shift Date: Load the shift, then click the "Change Date" button next to the date field. Enter the correct date to move the shift's record.
  • Deleting a Shift: Load the shift you wish to remove, then click the "Delete Shift" button. You will be asked to confirm this action.

Entering Patient Billings

  • Click "Add Patient" to add a new row to the table.
  • Use the checkboxes for each patient to specify categories like "80+", "WSIB", "IFH", etc.
  • In the "Billing Codes & Units" section, start typing a billing code. An autocomplete list will appear to help you. Select a code or type it fully.
  • The "Units" field defaults to 1. Change it if necessary for the specific code.
  • Referred Procedures: If you perform a procedure (like a fracture reduction) and then refer the patient to a specialist, check the "Ref" box next to that specific billing code. This will automatically reduce the value of that code (and any attached premiums like E412/E413) by 25% to reflect the correct billing amount.
  • The OHIP total for each patient and the summaries at the top will update automatically as you enter data.

Viewing Summaries & Earnings

As you enter data, the application automatically calculates your earnings. You can view these calculations in different tabs for the currently selected shift:

  • Billing Summary Tab: Shows a simple summary of the total OHIP value billed, the number of patients, and the average billing per patient.
  • Shift AFA Earnings Tab: This is where you see the detailed breakdown of your actual take-home pay. It includes your base pay, per-patient earnings, and all shadow billing calculations for the shift.

Analytics Dashboard

The "Analytics" tab gives you powerful insights into your earnings over time.

  • Use the month selector at the top to choose the month and year you wish to analyze.
  • The dashboard will display your total earnings, shifts worked, averages, and a breakdown of patient types for that period.
  • The Billing Code Frequency table at the bottom shows which codes you bill most often, total units, and total value for the selected period.
  • Click "Export Report to CSV" to download a spreadsheet of all shifts from the selected month for your own records.

Data Management: Backup & Restore

Your data is valuable. Use the features in the "Backup & Restore" tab to keep it safe.

  • Backup All Data to File: This is the most important button for data safety. It saves your entire shift database into a single JSON file. Do this regularly and save the file to a secure location (like a cloud drive or external hard drive).
  • Restore from File: Use this if you move to a new computer or need to recover from a data loss event. It will completely overwrite your current database with the data from the backup file you select.
  • Import Single Shift from File: This is useful for adding old shift files saved with a previous version of this app. It will add one shift to your database.
